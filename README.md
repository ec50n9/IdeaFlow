# IdeaFlow

一个基于无限画布的 AI 思维工作流工具。把碎片想法扔上白板，选中一批卡片，让不同模型帮你整理、扩写、演绎或转化。整个思考过程留在画布上，可以回溯、分叉、对比，看清自己是怎么从 A 走到 B 的。

> 专为这种人设计：脑子里有东西在乱跑，但整理不起来；需要让 AI 做辅助，又不想被对话框的线性流程绑住。

## 特性

- **无限画布**：自由放置卡片，拖拽连线，思维怎么走就怎么画。
- **Action 引擎**＜预配置好的 AI 任务一键触发。支持文生文、文生图、图生图、代码执行多种处理器。
- **多模型集群**：同时接入 GPT、Claude、Gemini、MiniMax 等多个供应商，不同阶段用不同模型。
- **次抛模式**：不想配置 action？选中卡片后临时填写提示词、选模型，立即执行，提示词本身也会成为画布上的一个节点。
- **多选与排序**：批量选中卡片，支持弹窗预排序后再传入 action，确保 AI 看到的顺序符合你的意图。
- **完整的过程留存**：每次 AI 调用都生成新节点，资源节点、任务轨迹、输出结果全部可见。

## 这不是自动化工具

n8n、扣子（Coze）、Dify 等工具的核心目标是 **让机器自动执行任务**。它们的节点是函数、API、条件判断，连线是数据流，画布上的是一套确定性的程序逻辑。配置好之后，人退出环路，机器照着跑。

IdeaFlow 做的是另一件事：**记录人的思维过程**。

一个核心原则：**画布上的卡片只是结果，提示词和处理过程都在 action 中**。你看到的每一张卡片都是产物，它背后的生成逻辑、使用的模型、输入的素材组成——这些都是 action 的责任。这样设计的好处是，你可以随时回到任意一个结果，看清它是怎么来的，然后用相同或不同的 action 重新演绎。

| | 自动化工作流 (n8n / 扣子 / Dify) | IdeaFlow |
|---|---|---|
| **节点是什么** | 功能模块（发邮件、调 API、判断分支） | 想法、素材、中间产物 |
| **连线是什么** | 数据流向 | 思维关联 |
| **画布是什么** | 程序逻辑图 | 思维地图 |
| **人在哪里** | 配置时在，运行时走开 | 每一步都由人触发和审视 |
| **运行方式** | 预编排的、确定性的自动执行 | 探索性的、非线性的、随时分叉回溯 |
| **产出什么** | 执行结果（发了邮件、写入了数据库） | 思维过程的可视化记录 |

简单说：工作流工具是你给机器下的指令集；IdeaFlow 是机器坐在你旁边，看着你在白板上想事情，你指哪它打哪。

## 谁适合，谁不适合

**适合**

- 脑子里有东西在乱跑，需要一块白板把碎片倒出来再整理的人。
- 写作、研究、分析时，需要不断分叉、比较、试错，而不是一条路走到黑的人。
- 想看清自己是怎么从 A 走到 B 的，而不只是要一个答案的人。
- 需要多个模型参与同一个问题，想要对比它们各自说了什么的人。

**不适合**

- 需要每天凌晨 8 点自动拉取数据、生成报表、发送邮件的人——这是 n8n 的活。
- 流程高度标准化，人不需要做任何判断，只要机器照着 SOP 跑的场景。
- 期望一个确定性的答案，不接受过程中随时变化、分叉、回退的人。

如果你的问题是"如何让这件事自动发生"，请用自动化工具。如果你的问题是"我该怎么想清楚这件事"，才用 IdeaFlow。

## 用法举例

**碎片整理机**

随手在白板上扔想法（语音转文字、复制粘贴、截图），不加整理。攒到七八张后选中，用一个"归纳" action 自动提取主题、去重、生成结构清晰的总结卡片。适合睡前/开会后清脑子。

**写作流水线**

左边一列素材卡片（引用、数据、灵感），中间是 Action 节点（大纲生成 → 段落扩写 → 润色），右边出成品。每个阶段不满意就分叉——从大纲节点另起一个分支换种风格写。整个思考过程全留在画布上，回头能看清自己是怎么从 A 走到 B 的。

**多模型辩论台**

同一份材料复制三份，分别丢给 GPT、Claude、Gemini 做分析（配置三个不同模型的 action），三个结果卡片并排显示。再用一个"综合研判" action 让第四个模型对比三份输出，生成最终结论。适合决策前的交叉验证。

**视觉探索链**

先文生图得到概念稿，把图卡片和提示词卡片连起来，复制几份改提示词做变体，选中一批做风格统一或细节修改（图生图）。最后用一个 action 生成设计说明或技术规格。做 UI 概念、角色设计、海报方案时很顺手。

**代码推演画布**

写算法或数据处理逻辑时，把输入数据、中间步骤、预期输出做成卡片，中间接 Code Action（工作线程执行环境）。可以一步一步跑，看每步的数据变化，错了就在中间插一张调试卡片。比 Jupyter 更自由，比纯文本更直观。

## 快速开始

**环境要求**

- Node.js 18+
- 一个 AI 提供商的 API Key（支持 OpenAI、Anthropic、Google、MiniMax 等）

**安装**

```bash
git clone git@github.com:ec50n9/IdeaFlow.git
cd IdeaFlow
npm install
```

**配置**

复制环境变量文件：

```bash
cp .env.example .env.local
```

在 `.env.local` 中填入至少一个 API Key：

```env
GEMINI_API_KEY=your_key_here
```

**启动**

```bash
npm run dev
```

打开 <http://localhost:3000> 即可使用。

**构建**

```bash
npm run build
```

## 技术栈

- **前端**：React 19 + TypeScript + Vite
- **画布引擎**：@xyflow/react
- **样式**：Tailwind CSS v4 + shadcn/ui + motion
- **状态管理**：Zustand
- **代码执行**：Web Worker
- **AI 调用**：多提供商适配器架构，支持 Stream / 文生图 / 图生图

## Roadmap

- [x] 支持生成图片和修改图片。
- [ ] 卡片左上角的 action tag，在生成模式为 code 模式的时候也需要显示为代码生成。action 配置中心中如果 action 是代码配置的也需要显示专门的提示。
- [ ] 单个卡片可以进入划线提取模式，方便用户手动提取关键内容。
- [ ] 双击卡片进入编辑模式时，卡片大小与原卡片不匹配，会马上缩小宽度，很突兀，需修复。
- [x] 用户选中卡片后，除了已配置的 action 以外，还可以进行自定义调用：弹窗选择模型、填写提示词、立即执行，即"次抛"模式。此模式下，用户输入的提示词会被创建为一个新节点，并与结果节点建立连接。
- [ ] action 配置页面增加"用户预排序"开关：仅对支持多选的 action 可见；开启后，点击该 action 时弹窗让用户排序，再将排序后的列表传入执行；若实际只选中了一项，则跳过弹窗。
- [ ] 支持同一个 action 切换不同模型来做。
- [ ] action 不再显示为卡片左上角的 tag，而是独立显示为画布上的中间节点（小型卡片）。所有源卡片连接到该中间节点，中间节点再连接到生成的结果卡片。选中中间节点可以拷贝并重新运行该 action；同 action 切换模型的功能也在此节点上交互。
- [/] action 中不再固定指定模型，而是改为设置需要使用的模型能力（如图生图、文生文等）。执行 action 时根据能力需求匹配当前已配置的模型插槽。code 模式下可动态添加多个模型插槽，每个插槽有唯一标识和对应的能力配置，代码中通过引用唯一标识来调用模型。在画布上触发 action 时，需要先根据插槽选择执行模型，然后才开始执行。
- [ ] action 的激活条件支持更多选项，比如限制文本卡片、图片卡片等。
- [x] 当模型协议使用 Response API 协议的时候，要求选择 Responses 图像模型，目前只支持 gpt-image-2 模型（也就是现在我写死默认值到 tools 里面的那个）
- [ ] 图片生成增加 size、n、quality、aspect_ratio 等字段支持，Images API 不支持直接传入参数，可以把尺寸与比例直接写进 prompt，例如：
  - 横版 16:9 电影画幅，黄昏时的海边老灯塔
  - 竖版 9:16 手机壁纸，赛博朋克城市雨夜
  - 1024×1024 方形 LOGO，极简猫咪线条
  建议把尺寸描述放在 prompt 最前面，模型遵循度更高。
